    <div class="accordion">
      <div class="theory-grid">
        <div class="theory-card">
          <div class="theory-title">@RestController</div>
          <div class="theory-body"><code>@Controller</code> + <code>@ResponseBody</code>. Mọi method return value được serialized to JSON/XML. <code>@Controller</code> dùng khi trả về View (Thymeleaf). <code>@RequestMapping("/api/v1")</code> cho base path.</div>
        </div>
        <div class="theory-card">
          <div class="theory-title">HTTP mapping annotations</div>
          <div class="theory-body"><code>@GetMapping</code> (read), <code>@PostMapping</code> (create), <code>@PutMapping</code> (full update), <code>@PatchMapping</code> (partial update), <code>@DeleteMapping</code> (delete). Params: <code>@PathVariable</code>, <code>@RequestParam</code>, <code>@RequestBody</code>.</div>
        </div>
        <div class="theory-card">
          <div class="theory-title">ResponseEntity&lt;T&gt;</div>
          <div class="theory-body">Kiểm soát HTTP status + headers + body. <code>ResponseEntity.ok(body)</code> = 200. <code>ResponseEntity.created(uri).body(obj)</code> = 201. <code>ResponseEntity.noContent().build()</code> = 204. <code>ResponseEntity.notFound().build()</code> = 404.</div>
        </div>
        <div class="theory-card">
          <div class="theory-title">Request handling</div>
          <div class="theory-body"><code>@PathVariable Long id</code>: <code>/students/{id}</code>. <code>@RequestParam(required=false) String name</code>: <code>/students?name=Long</code>. <code>@RequestBody StudentDto dto</code>: JSON body → object (via Jackson). <code>@Valid</code> kết hợp với <code>@RequestBody</code>.</div>
        </div>
      </div>
    </div>

    <!-- Code -->
    <div class="code-wrap">
      <button class="copy-btn">Copy</button>
      <pre>@RestController
@RequestMapping("/api/v1/students")
public class StudentController {

    private final StudentService service;

    public StudentController(StudentService service) {
        this.service = service; // Constructor injection
    }

    @GetMapping
    public ResponseEntity&lt;List&lt;StudentDto&gt;&gt; getAll() {
        return ResponseEntity.ok(service.findAll());
    }

    @GetMapping("/{id}")
    public ResponseEntity&lt;StudentDto&gt; getById(@PathVariable Long id) {
        return service.findById(id)
            .map(ResponseEntity::ok)
            .orElse(ResponseEntity.notFound().build());
    }

    @PostMapping
    public ResponseEntity&lt;StudentDto&gt; create(@Valid @RequestBody StudentDto dto) {
        StudentDto created = service.create(dto);
        URI location = URI.create("/api/v1/students/" + created.getId());
        return ResponseEntity.created(location).body(created);
    }

    @PutMapping("/{id}")
    public ResponseEntity&lt;StudentDto&gt; update(
            @PathVariable Long id,
            @Valid @RequestBody StudentDto dto) {
        return ResponseEntity.ok(service.update(id, dto));
    }

    @DeleteMapping("/{id}")
    public ResponseEntity&lt;Void&gt; delete(@PathVariable Long id) {
        service.delete(id);
        return ResponseEntity.noContent().build();
    }
}</pre>

        </div>
        <button class="qa-toggle">Xem đáp án ▼</button>
        <div class="qa-answer"><code>@PathVariable</code>: resource identifier trong URL path (<code>/students/42</code>). <code>@RequestParam</code>: optional filter/sort trong query string (<code>/students?sort=name&amp;page=1</code>).       <div class="qa-item">
        <div class="qa-header">
          <span class="qa-badge" style="background:#E67E22">Trung</span>
          <span class="qa-q">HTTP status codes quan trọng trong REST API — 200, 201, 204, 400, 401, 403, 404, 409, 500.</span>
        </div>
        <button class="qa-toggle">Xem đáp án ▼</button>
        <div class="qa-answer">200 OK (GET/PUT success), 201 Created (POST with Location header), 204 No Content (DELETE), 400 Bad Request (validation fail), 401 Unauthorized (no auth), 403 Forbidden (no permission), 404 Not Found, 409 Conflict (duplicate), 500 Internal Server Error.</div>

      <div class="qa-item">
        <div class="qa-header">
          <span class="qa-badge" style="background:#E67E22">Trung</span>
          <span class="qa-q">Tại sao nên dùng Constructor Injection thay vì <code>@Autowired</code> field injection?</span>
        </div>
        <button class="qa-toggle">Xem đáp án ▼</button>
        <div class="qa-answer">Constructor: dependencies visible (explicit contract), immutable (final fields), testable (<code>new MyClass(mock)</code>), no reflection needed. Field injection: hidden dependencies, requires Spring context for testing, allows mutable state. Spring team recommends constructor injection. IntelliJ warns about field injection.</div>
      </div>

        <div class="qa-header">
          <span class="qa-badge" style="background:#E67E22">Trung</span>

        </div>
        <button class="qa-toggle">Xem đáp án ▼</button>
        <div class="qa-answer">Jackson ObjectMapper convert JSON→Java. Spring Boot auto-configures Jackson. Customize via <code>@JsonProperty</code>, <code>@JsonIgnore</code>, <code>@JsonFormat</code>. Global config: <code>JacksonConfig</code> bean với <code>ObjectMapper</code> setup (<code>FAIL_ON_UNKNOWN_PROPERTIES=false</code>, date format, snake_case). <code>spring.jackson.*</code> properties.</div>
      </div>

        </div>
        <button class="qa-toggle">Xem đáp án ▼</button>
        <div class="qa-answer">(1) URI: <code>/api/v1/students</code> — visible, cacheable, easy to route. (2) Header: <code>Accept: application/vnd.app.v1+json</code> — clean URI, hard to test in browser. (3) Query param: <code>/students?version=1</code> — easy, not REST-ful. Production: URI versioning phổ biến nhất (AWS, Twitter). Chú ý: không version minor changes, chỉ version breaking changes.</div>

        <div class="qa-header">
          <span class="qa-badge" style="background:#C0392B">Khó</span>

        </div>
        <button class="qa-toggle">Xem đáp án ▼</button>
        <div class="qa-answer">Load 100 students (1 query). Access <code>student.getCourse()</code> in loop → 100 separate queries = 101 total. Fix: <code>@Query("SELECT s FROM Student s JOIN FETCH s.course WHERE ...")</code> → 1 query with JOIN. Or <code>@EntityGraph(attributePaths={"course"})</code> on repository method.         <div class="sq-item">
          <div class="sq-q">What is Spring Boot auto-configuration?</div>
          <div class="sq-a">Spring Boot detects dependencies on classpath and auto-configures beans. If spring-data-jpa on classpath → configures DataSource, EntityManagerFactory, TransactionManager. Controlled by @ConditionalOnClass/@ConditionalOnMissingBean. Override by declaring own @Bean. View in actuator /conditions endpoint.</div>
        </div>

        <div class="sq-item">
          <div class="sq-q">Difference between @Component, @Service, @Repository?</div>
          <div class="sq-a">All are @Component specializations, all eligible for component scanning. @Repository adds PersistenceExceptionTranslation AOP (converts DB exceptions to DataAccessException). @Service marks business layer (transaction boundary). Semantic meaning helps readability and AOP targeting.</div>
        </div>

        <div class="sq-item">
          <div class="sq-q">What is Spring IoC container?</div>
          <div class="sq-a">ApplicationContext is the IoC container — manages bean lifecycle, DI, AOP, events. Beans registered via @Component scan or @Bean methods. Container injects dependencies, applies proxies (@Transactional, @Async). Singleton scope default — one instance shared. Prototype — new instance per request.</div>
        </div>
      </div>

      <div class="sq-group">
        <div class="sq-group-title">REST API + JPA</div>

        <div class="sq-item">
          <div class="sq-q">How to handle validation errors globally in Spring Boot?</div>
          <div class="sq-a">@RestControllerAdvice with @ExceptionHandler(MethodArgumentNotValidException.class). Extract field errors from BindingResult. Return ResponseEntity with 400 status and error map. RFC 7807 Problem Details format recommended for consistency.</div>
        </div>

        <div class="sq-item">
          <div class="sq-q">Spring Data JPA derived query vs @Query — when to use each?</div>
          <div class="sq-a">Derived (findByNameAndGpaGreaterThan): simple conditions, auto-generated, readable. @Query JPQL: complex joins, groupBy, subqueries. @Query nativeQuery=true: DB-specific SQL, performance-critical. Rule: start with derived, switch to @Query when derived gets complex (&gt;3 conditions).</div>
        </div>

        <div class="sq-item">
          <div class="sq-q">What is JPA N+1 problem and how to fix?</div>
          <div class="sq-a">Loading N entities then accessing lazy collection triggers N additional queries. Fix: JOIN FETCH in JPQL, @EntityGraph for dynamic loading, DTO projection to select only needed fields. Detect with: spring.jpa.show-sql=true or Hibernate statistics.</div>
        </div>
      </div>

      <div class="sq-group">
        <div class="sq-group-title">Spring Production Patterns</div>

        <div class="sq-item">
          <div class="sq-q">How do Spring Profiles work?</div>
          <div class="sq-a">@Profile("dev") beans only active when dev profile active. Activate via spring.profiles.active=dev (property), SPRING_PROFILES_ACTIVE=dev (env var, highest priority), @ActiveProfiles("test") in tests. Environment-specific application-{profile}.yml files. Use for: different DB configs, mock vs real services, feature flags.</div>
        </div>

        <div class="sq-item">
          <div class="sq-q">@Transactional self-invocation problem?</div>
          <div class="sq-a">Spring @Transactional uses AOP proxy. External call → proxy → transaction begins. Internal call (this.method()) → bypasses proxy → no transaction. Fix: move to separate @Component bean or use ApplicationContext.getBean(this.getClass()) (antipattern). Best: design to avoid internal calls.</div>
        </div>

        <div class="sq-item">
          <div class="sq-q">What does Spring Actuator provide?</div>
          <div class="sq-a">Production-ready monitoring endpoints. /health: app health + DB/Redis connectivity (use for K8s liveness/readiness probes). /metrics: JVM, HTTP, custom. /info: build info. /loggers: change log level at runtime. Secure in production: only expose /health publicly.</div>
        </div>
