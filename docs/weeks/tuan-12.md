          <li><em>"What is the N+1 problem?"</em> → When fetching N parent entities triggers N extra queries for their children; solved with a fetch join or an entity graph.</li>
        </ul>

record Employee(Long id, String name, double salary) {
    // record tự sinh equals/hashCode/toString chuẩn theo mọi field

    public static void main(String[] args) {
        List&lt;Employee&gt; emps = List.of(

            .collect(Collectors.toList());

(3) <strong>Bill Pugh / static holder</strong> — dùng inner static class, JVM đảm bảo lazy + thread-safe khi load holder, cách được khuyên dùng nhất. import java.util.*;

class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map&lt;Integer, Integer&gt; seen = new HashMap&lt;&gt;();
        for (int i = 0; i &lt; nums.length; i++) {

            }
            seen.put(nums[i], i);
        }
        return new int[]{}; // theo đề sẽ không tới đây
    }

            <span class="qa-q">Thiết kế hệ thống rút gọn URL (URL shortener) ở mức cao?</span>

(3) Design: API POST /shorten + GET /{code}; sinh code bằng base62 encode của ID tự tăng (hoặc hash + chống trùng); lưu mapping trong DB (key-value hợp với NoSQL hoặc SQL có index). (4) Tối ưu: cache hot URL trong Redis để redirect cực nhanh; CDN; read replica. (5) Tradeoff: counter-based đơn giản nhưng lộ thứ tự; hash-based cần xử lý collision.
